# Claude Code — Standard Project Setup

This document describes the standard file layout and configuration approach for all our projects using Claude Code. Copy it to new projects and adjust the project-specific sections.

---

## File Map

```
project-root/
├── .mcp.json                        # Project MCP servers — committed to git (cross-platform only)
├── CLAUDE.md                        # AI instructions — committed to git
├── .claude/
│   ├── settings.json                # Shared project AI settings — committed to git
│   ├── settings.local.json          # Machine-local AI settings — gitignored
│   ├── agents/                      # Custom subagent definitions — committed to git
│   └── skills/                      # Custom skill definitions — committed to git
└── docs/
    └── CLAUDE/                      # Extended AI context docs — committed to git
        ├── architecture.md
        ├── conventions.md
        └── features.md

~/.claude/
└── settings.json                    # Global user settings (all projects)

~/.claude.json                       # Claude Code internal state + local project MCP servers
```

### What goes where

| File | Scope | Committed | Purpose |
|------|-------|-----------|---------|
| `.mcp.json` | Project | Yes | MCP servers safe to share (cross-platform commands only) |
| `CLAUDE.md` | Project | Yes | Rules and context Claude reads on every session |
| `.claude/settings.json` | Project | Yes | Shared permissions, hooks, plugin enablement |
| `.claude/settings.local.json` | Project | **No** | Machine-specific permissions/hooks |
| `~/.claude/settings.json` | Global | N/A | User-level defaults (model, plugins, etc.) |
| `~/.claude.json` | Global | N/A | Internal state; also stores local-scoped MCP servers per project |

## MCP Servers

### Strategy

- **`.mcp.json`** — only servers whose launch commands work identically on all platforms (e.g. `php artisan`).
- **`~/.claude.json` (local scope)** — platform-specific servers (e.g. `cmd /c npx ...` on Windows vs `npx ...` on Mac/Linux). Added via `claude mcp add` from within the project directory.

### Universal servers (`.mcp.json`)

These belong in `.mcp.json` for every Laravel project:

```json
{
    "mcpServers": {
        "laravel-boost": {
            "command": "php",
            "args": ["artisan", "boost:mcp"]
        }
    }
}
```

### Project-specific local servers

The servers below are used in **this project** but may not apply to others. Each developer adds them locally (not committed).

#### macOS / Linux

```bash
claude mcp add shadcn   -- npx shadcn@latest mcp
claude mcp add magicui  -- npx -y @magicuidesign/mcp@latest
claude mcp add context7 -- npx -y @upstash/context7-mcp@latest
```

#### Windows (Git Bash / MSYS2)

```bash
claude mcp add shadcn   -- cmd /c npx shadcn@latest mcp
claude mcp add magicui  -- cmd /c npx -y @magicuidesign/mcp@latest
claude mcp add context7 -- cmd /c npx -y @upstash/context7-mcp@latest
```

> Run these from inside the project directory. The `--scope local` default stores them in `~/.claude.json` under the project entry — never committed.

### Other MCP servers to consider for new projects

| Server | Command (Mac/Linux) | Notes |
|--------|---------------------|-------|
| `context7` | `npx -y @upstash/context7-mcp@latest` | Library docs lookup — useful in most projects |
| `shadcn` | `npx shadcn@latest mcp` | Only for projects using shadcn/ui |
| `magicui` | `npx -y @magicuidesign/mcp@latest` | Only for projects using Magic UI |
| Claude.ai Slack | (via claude.ai connector) | Useful for all projects |
| Claude.ai ClickUp | (via claude.ai connector) | Useful for all projects |

## CLAUDE.md Structure

Keep the top section short and project-specific. The `<laravel-boost-guidelines>` block below the `===` separator is auto-generated by `composer run post-update-cmd` — do not edit it manually.

```markdown
# Project Name

**Status:** <development/production>

**Stack:** Laravel X / Inertia v2 / React 19 / TypeScript / Tailwind v4 / ...

**Rules:** <key project-specific rules>

**Docs:** [Architecture](docs/CLAUDE/architecture.md) · [Conventions](docs/CLAUDE/conventions.md)

===

<laravel-boost-guidelines>
...auto-generated...
</laravel-boost-guidelines>
```

### docs/CLAUDE/ files

Break extended context out of `CLAUDE.md` into topic files. Keep `CLAUDE.md` under ~15 lines before the `===`. Typical files:

| File | Contains |
|------|----------|
| `architecture.md` | Routing layout, controller namespaces, page structure, shared data |
| `conventions.md` | Naming, patterns, component rules specific to this project |
| `features.md` | Domain features, business rules, access control matrix |

## .claude/settings.json (shared)

Minimal example — only add what the whole team needs:

```json
{
  "permissions": {
    "allow": ["mcp__*"]
  }
}
```

## .claude/settings.local.json (gitignored)

Machine-specific permissions. This project includes ready-made example files in `.claude/`:

- `.claude/settings.local.json.mac.example` — recommended for macOS/Linux
- `.claude/settings.local.json.windows.example` — recommended for Windows

When setting up a new project or configuring an existing one, copy the appropriate file:

```bash
# macOS / Linux
cp .claude/settings.local.json.mac.example .claude/settings.local.json

# Windows (Git Bash)
cp .claude/settings.local.json.windows.example .claude/settings.local.json
```

Ensure `.claude/settings.local.json` is in `.gitignore`.

## ~/.claude/settings.json (global)

User-level defaults that apply to all projects. Edit directly — not project-specific.

```json
{
  "model": "sonnet",
  "autoUpdatesChannel": "latest",
  "enabledPlugins": {
    "superpowers-laravel@superpowers-laravel-marketplace": true,
    "frontend-design@claude-plugins-official": true
  }
}
```

## .claude/agents/ and .claude/skills/

- **agents/** — subagent markdown files (e.g. `frontend-reviewer.md`, `test-runner.md`). Committed; available to the whole team.
- **skills/** — skill override files that activate domain-specific instructions (e.g. `pest-testing`, `tailwindcss-development`). Committed.

Create new agents/skills with the `/create-agent` and `/create-skill` Claude commands.

## AGENTS.md — AI Agent Runbook

`AGENTS.md` in the project root is a practical runbook for AI coding agents (e.g. Claude, Codex, Gemini CLI). It covers environment setup, starting/stopping the dev server, running tests, and a task completion checklist.

### Agent server scripts

`scripts/agent-serve.sh` and `scripts/agent-stop.sh` provide a consistent way for any agent to spin up and tear down a local dev server without conflicting with other processes.

- `agent-serve.sh [port]` — starts `php artisan serve` in the background on the given port (default 8001), auto-incrementing if the port is taken. Writes `.agent-server.pid` and `.agent-server.env`.
- `agent-stop.sh` — kills the process from `.agent-server.pid` and removes both files.

Both files are gitignored. Make the scripts executable on first checkout:

```bash
chmod +x scripts/agent-serve.sh scripts/agent-stop.sh
```

Usage:

```bash
bash scripts/agent-serve.sh          # start on 8001 (or next free port)
source .agent-server.env             # load SERVER_PORT into shell
curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:$SERVER_PORT/
bash scripts/agent-stop.sh          # stop and clean up
```

## Checklist for a new project

- [ ] Add `.mcp.json` with `laravel-boost` (and any other cross-platform servers)
- [ ] Create `CLAUDE.md` — top section only; run `composer update` to populate `<laravel-boost-guidelines>`
- [ ] Create `docs/CLAUDE/architecture.md`, `conventions.md`, `features.md`
- [ ] Add `.claude/settings.json` with shared permissions
- [ ] Copy `.claude/settings.local.json.mac.example` or `.windows.example` to `.claude/settings.local.json` (gitignored)
- [ ] Run `claude mcp add` for project-specific local servers (per OS)
- [ ] Confirm `~/.claude/settings.json` has preferred global model and plugins
- [ ] Add `scripts/agent-serve.sh` and `scripts/agent-stop.sh` and create `AGENTS.md`
- [ ] Add `.agent-server.pid` and `.agent-server.env` to `.gitignore`
